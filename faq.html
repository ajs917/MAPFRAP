What tools have you used to build your website and how did you selected them?

BootStrap:
The template for this website was found on Start Bootstrap. The ‘Modern Business’ template was used. The template was utilised due to its appropriate configuration to employ cowboy coding principles for the required content of the website. For example, this section was well suited to the FAQ page and the Contact Us page had a validated form. Additionally, the template was free.

GitHub Desktop:
GitHub was used to host and publish the website. There are many benefits to using GitHub – its free, collaborative, implements version control and is simple to use for anyone. A repository was created that contains all the contents of the webpage, and GitHub desktop was used to push and pull commits as required when changes were made to the source code.

Atom Editor:
The editor used for website design was Atom Editor. Atom is a modern free text editor by GitHub. The software can be integrated wit GitHub to commit changes directly from the Atom interface – making it the ideal choice for editing.

Describe the steps to register and setup a domain name for your website

1) Select a domain name:
Key things to consider when selecting the domain name include:

  • Buy both the .co.uk and .com versions of your domain name to avoid confusion to users
  • Use keywords for your domain name, in this case the name of the product/company makes sense
  • Keep domain name short and memorable, avoid use of numbers and characters where possible

2) Check the domain name is available:

  • Check the domain name is not trademarked – this can be done on the Government Website. It may be worth cross checking across multiple domains to avoid legal trouble.
  • Check your domain name has not been taken. Nominet is the official registry of UK domain names and runs a service called WhoIs to check domain names.

3) Register the domain name using a registrar

  • Nominet keeps a list of official registrars. Anyone in the list can be used, and the process of registering varies by registrar.
  • Some registrars are accredited – it is recommended to use accredited registrars like GoDaddy, WebWiz, Google Domains etc.
  • Follow through the process of the registrar and purchase your required domain. Several top-level domains (org, co.uk, com, biz, net etc.) may be available, select as many as required.

4) Purchase the rights to the domain name

  • Select how long to keep the rights to the domain name
  • Pay a fee to the registrar to make the registration
  • Once the transaction is complete, you own the domain name

5) Link the domain to your website

  • Specify the IP address where users go to when using your domain. The process to do this varies depending on your registrar and host server (i.e GitHub Pages)
  • GitHub Pages has an option to set a custom domain under settings of the repository – input your register domain name in the Custom Domain field. This adds a CNAME file to source files
  • Configure your Domain Name System (DNS) by creating a CNAME record. This process varies depending on your registrar, but once this is done your domain should be properly configured

What do you feel makes a good website?

There are too many factors that make a good website to list, but the most important factors are highlighted below:

User Interface Design:
User interface is how the user interacts with the website. The best websites have user interface at the heart of their design process. There are several key elements to UI Design. Consistency of pages, pleasing aesthetics and intuitive operation are some of the features of a good user interface. Additional features include efficiency of navigation

User Experience Design:
User Experience Design, while largely like User Interface Design, has key differences. Firstly, UX focuses on optimising a user’s experience of the website, whereas UI focuses on how a user interacts with the website. As such elements of UX that make a great website include minimalism – in design and in content. Keeping layouts uncluttered is key to improving user experience. Additionally, users want to be able to find the information they want without navigating/reading through piles of unnecessary information. Other elements include speed of content loading, frequency of pop up ads and variable colour schemes/text sizes.

Responsive Design:
With phones, tablets and laptops all become commonplace, the best websites are ones that respond to the different screen sizes. Responsive design means designing the website to ensure that users can access the website in an appealing and intuitive way on all screen sizes. Bootstrap templates are all created with responsiveness in mind – utilising features such as fluid grids and flexible layouts. It can be argued that responsive design is a part of UI Design, while this may be true it is important enough to get its own mention.

What is accessibility and what considerations should you make for it?

Accessibility in web design terms means that websites are designed so that individuals regardless of any impairment/handicap can still utilise the facilities. Accessibility is an extremely important consideration in User Experience, as user should have equal opportunity to access the information at hand.

A few examples of accessibility considerations include:
  1. Options for increasing/decreasing font sizes to assist in readability
  2. Including alt text in images to ensure the gist of images can still be appreciated if they don’t load
  3. Options to alter colour schemes to assist with impairment due to colour blindness
  4. Navigation with just the keyboard in the event of a mouse not being used
What is GDPR and what considerations should be made for privacy on your website?

The General Data Protection Regulation is a regulation in the EU on data protection and privacy. The law regulates how data is processed, stored and transferred. GDPR has impact on websites, as summarised below

# Principle Considerations
1 Lawfulness, Fairness and Transparency Data must be obtained in a lawful, transparent and fair way. The only data being collected is through a users own submission in the contact form - consent should also be recorded. The information on team members was obtained and utilised with permission.
2 Purpose Limitation Data collected should be for a specific reason. In this website, information is only being collected in order to respond to queries through the contact form. The information is only used to respond to queries.
3 Data Minimisation The information collected must be relevant and limited to the purpose at hand. The contact form only asks for information which is pertinent to effectively dealing with queries.
4 Accuracy Data held should be accurate and if necessary, up to date. Data collected from the form should be accurate to the best of our ability - given the information is provided from users this is expected. The information provided by individual team members is accurate and will be updated regularly to ensure compliance.
5 Storage Limitation Data should only be stored for an appropriate amount of time, and disposed of securely after this retention period has passed. In this instance, should a query be dealt with and the customer satisified, their data should be securely disposed of shortly after. With regard to team members, should a team member resign, their information should also be securely disposed.
6 Integrity and Confidentiality Personal data should be treated with privacy and respect. For this website, data provided will not be distributed or used for purposes other than responding to the query submitted. Spam mails or selling of data will not occur, and every effort to minimise security breaches will be taken.
7 Accountability The controller has responsiblity for compliance to GDPR. As we, the business determine the purposes of the data we process, we are the data controllers and as such are responsible for ensuring practices are in line with GDPR regulations.
What marketing steps would you take to drive people to look at your website?

In order to increase the impact of the website several marketing techniques could be employed. First and foremost, search engine optimisation is pivotal. SEO means increasing the likelihood of appearing higher up the returned results list in a search engine. Individuals often ‘just Google’ information they are interested in and showing up within the first two pages greatly improves the popularity of the website. Search engine optimisation is a complex area which involves intricate knowledge of how search engines work. Indexing the website in popular search engines is a good start. Identifying keywords and utilising them in your source files meta data may also assists in SEO, although less effectively nowadays. Moz provide good information on SEO.

Additionally, the power of social media is incredible for increasing the presence of a website. Setting aside a budget for sponsored promotion on the likes of Instagram, Facebook and Snapchat will greatly increase the visibility of the website and encourage readership.
